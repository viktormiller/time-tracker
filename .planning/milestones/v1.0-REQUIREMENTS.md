## Version 1: Deployment MVP + Enhancements

### REQ-001: Single User Authentication
**Priority:** High
**Status:** Complete
**Outcome:** JWT authentication with session management, refresh tokens, CSRF protection. Architecture supports future multi-user expansion.

### REQ-002: Docker Containerization
**Priority:** High
**Status:** Complete
**Outcome:** Separate Dockerfiles for frontend (nginx:alpine) and backend (node:22-slim). Multi-stage builds, health checks, non-root execution.

### REQ-003: Docker Compose Orchestration
**Priority:** High
**Status:** Complete
**Outcome:** Full docker-compose.yml with service dependencies, Docker Secrets, network isolation. Single `docker-compose up` launches entire stack.

### REQ-004: Tempo Issue Key Resolution
**Priority:** Medium
**Status:** Complete
**Outcome:** Tempo entries display Jira issue keys (e.g., "ABC-27"). Clickable links to Jira. Fallback to internal ID when key unavailable.

### REQ-005: PostgreSQL Database Migration
**Priority:** High
**Status:** Complete
**Outcome:** PostgreSQL replaces SQLite in production. TIMESTAMPTZ for timezone support. pgloader migration script. Database-level UUID generation with gen_random_uuid().

### REQ-006: Light/Dark Theme Toggle
**Priority:** Medium
**Status:** Complete
**Outcome:** Theme toggle with sun/moon icons. localStorage persistence. FOUC prevention. Complete dark mode styling across all components including charts.

### REQ-007: CSV Export
**Priority:** Medium
**Status:** Complete
**Outcome:** Client-side CSV generation with German column headers. Formula injection sanitization. Date range in filename. Alert on empty export.

### REQ-008: PDF Export
**Priority:** Low
**Status:** Complete
**Outcome:** Server-side Puppeteer PDF generation. Chart + table layout. German locale formatting. System Chromium in Docker.

### REQ-009: Robust Synthetic ID Generation
**Priority:** Medium
**Status:** Complete
**Outcome:** Database-level gen_random_uuid() in PostgreSQL. No duplicate IDs across sources. Concurrent sync safe via unique constraints.

### REQ-010: Manual Time Entry
**Priority:** High
**Status:** Complete
**Outcome:** AddEntry form with react-hook-form + zod validation. Duration from start/end time. Project autocomplete. "MANUAL" source tag.

## Version 2: CLI Tool

### REQ-011: Go CLI - Today's Hours
**Priority:** Medium
**Status:** Complete
**Outcome:** `timetracker today` shows total hours with source breakdown. JWT authentication. Config at ~/.timetracker/config.yaml.

### REQ-012: Go CLI - Weekly Summary
**Priority:** Medium
**Status:** Complete
**Outcome:** `timetracker week` shows daily/source breakdown as ASCII table. Custom display package for formatting.

### REQ-013: Go CLI - Trigger Sync
**Priority:** Medium
**Status:** Complete
**Outcome:** `timetracker sync` triggers all provider syncs via POST /api/sync. Optional force flag.

## Version 3: Future Sources (Deferred)

### REQ-014: Clockify Integration
**Priority:** Low
**Status:** Deferred — removed from v1.0 roadmap, to be addressed in future milestone

### REQ-015: Clean Provider Abstraction
**Priority:** Medium
**Status:** Complete
**Outcome:** TimeProvider interface with sync(), validate(), getName(), getCachePath(). BaseTimeProvider abstract class. Adding new sources requires extending one interface.
